# Changelog

Per-package changelogs live in `packages/<name>/CHANGELOG.md`, maintained
automatically by `semantic-release` on every merge to `main` that includes a
semver-relevant commit (`feat` / `fix` / `perf` / `BREAKING CHANGE`).

This page renders the current contents at build time — Vercel rebuilds the
docs on every push, including the `chore(release): ...` commit that
`semantic-release` pushes back with `[skip ci]` (Vercel doesn't respect that
marker, only GitHub Actions does), so this view is always current with the
latest release.

## Format

Each entry is grouped by commit type. Example:

```markdown
## [1.4.0](https://github.com/...) (2026-07-10)

### Features

* **components:** add Toast with variants + auto-dismiss ([abc1234](https://...))

### Bug Fixes

* **components:** correct disabled opacity on Button ghost variant ([def5678](https://...))
```

Each version link points at the GitHub compare view between the previous tag
and this one, so you can see the exact commits that landed.
